# 로또아이 결과 공개 검증 백서
*(LOTTOi Public Verification Structure Whitepaper – KR)*

---

## 1. 문서 개요

로또아이는 회원이 생성한 번호를  
**추첨 이전 시점**에 외부 시스템에 기록하고,  
그 기록이 **사후에 변경되었는지 여부를 누구나 직접 확인할 수 있도록**  
공개 검증 구조로 운영됩니다.

본 문서는 알고리즘이나 예측 성능을 설명하기 위한 문서가 아니라,  
**기록의 생성 시점, 공개 방식, 변경 이력 확인 방법**을  
외부 기준으로 설명하기 위한 운영·검증 구조 문서입니다.

---

## 2. 공개 검증 구조의 기본 원칙

로또아이의 공개 검증 구조는 다음 원칙을 기반으로 합니다.

- 기록은 **추첨 이전 시점에 생성**
- 기록은 **외부 제3자 시스템에 남김**
- 기록 이후의 **변경 여부와 시점은 은폐 불가**
- 검증의 주체는 운영자가 아닌 **누구나**

로또아이는 기록의 수정을 원천적으로 차단하는 대신,  
**수정이 발생하더라도 그 사실과 시점을 숨길 수 없도록**  
구조적으로 설계합니다.

---

## 3. 공개 검증 전체 흐름 (실제 운영 기준)

[회차 Google 공개 문서 생성]
└ 최초 생성 시 읽기 전용(Viewer) 권한
└ 문서 생성 시점 Google 시스템 자동 기록
        ↓
[GitHub에 Google 문서 URL · 기록 시점 로그 저장]
└ URL 최초 기록 시점 외부 고정
└ 수정 발생 시 커밋 이력으로 변경 시점 공개
        ↓
[회원 번호 생성(추천/생성) 및 시스템 처리]
        ↓
[추첨 당일 (토요일) 20:00 ~ 20:30]
└ Google 문서 편집 권한 전환
└ 권한 변경 시점 또한 외부 시스템에 기록
        ↓
[Google 공개 문서에 회원 생성(추천/생성) 번호 저장]
└ 저장 시점 및 변경 이력 자동 기록
        ↓
[추첨 진행]
        ↓
[누구나 외부에서 생성 시점 · 권한 변경 · 변경 이력 검증]


---

## 4. GitHub 로그의 역할

GitHub 로그는 번호 내용이나 결과를 저장하기 위한 용도가 아닙니다.  
해당 회차의 **Google 공개 문서 URL이 언제부터 외부에 존재했는지**를  
가장 이른 시점에 고정하기 위한 **외부 기준 로그(anchor)** 역할을 합니다.

이를 통해 다음 사항을 확인할 수 있습니다.

- 해당 문서 URL이 **추첨 이후에 생성된 것이 아님**
- GitHub 기록 시점 이전에는  
  이미 외부에서 접근 가능한 문서가 존재했음

GitHub 문서는 필요 시 수정될 수 있으며,  
모든 수정은 **커밋 이력으로 시간 순서대로 공개**됩니다.

---

## 5. Google 문서 권한 운영 방식

### 5-1. 문서 최초 생성 시
- 읽기 전용(Viewer) 권한
- 사전 변경 차단
- 문서 생성 시점 자동 기록

### 5-2. 추첨 당일 (토요일 20:00 ~ 20:30)
- 회원 생성 번호 저장 직전
- 문서 편집 권한으로 전환
- 권한 변경 시점 또한 Google 시스템에 기록

이는 추첨 이전에  
로또아이 시스템이 수행한 모든 준비 절차가  
외부 시간 기준으로 남도록 하기 위한 설계입니다.

---

## 6. 수정 가능성과 변경 이력에 대한 기준

추첨 이후에는 Google 공개 문서의 편집 권한이 유지되며,  
운영자 또는 권한이 부여된 누구나 수정이 가능할 수 있습니다.

다만 이 경우에도:

- 변경 시점
- 변경 내용
- 변경 순서

는 Google 문서의 변경 이력에  
자동으로 기록되어 남습니다.

따라서 기록이 추첨 이후에 수정되었는지 여부는  
누구나 사후적으로 확인할 수 있으며,  
수정 사실이 은폐될 수 없습니다.

---

## 7. URL 오기입 등 사전 수정 발생 시 처리 원칙

운영상 오류(URL 오기입 등)로 인해  
추첨 이전에 GitHub 기록이 수정되는 경우가 발생할 수 있습니다.

이 경우에도 다음 정보는 모두 공개됩니다.

- 최초 기록 시점
- 수정 발생 시점
- 수정 전·후 URL

검증 시에는  
**최종 확정된 URL이 추첨 이전에 존재했는지 여부**와  
**수정이 추첨 이전에 이루어졌는지**를 기준으로 판단할 수 있습니다.

---

## 8. 외부 공개 검증 시 확인해야 할 핵심 요소

[외부 공개 검증 핵심 요소]

└ GitHub 저장소
· Google 문서 URL의 최초 기록 시점 확인
· 기록 시점이 추첨 이전인지 최종 시간 확인
· 커밋(변경) 이력으로 수정 여부 및 시점 확인

└ Google 문서
· 문서 생성 시점 및 변경 이력 확인
· 추첨 전 기준 시점(토요일 20:35 이전)의
최종 변경 내용 확인
· 해당 시점 기준 문서 다운로드 가능


---

## 9. 기술적 한계 및 운영 환경 고지

외부 네트워크 지연,  
Google/GitHub 플랫폼 장애,  
기타 기술적인 요인으로 인해  
기록 생성 또는 권한 변경 시점이  
일시적으로 지연될 수 있습니다.

이 경우에도 실제 생성 시점과 변경 이력은  
외부 시스템에 그대로 남아,  
추첨 시점과의 시간 관계를  
사후적으로 확인할 수 있도록 설계되어 있습니다.

본 구조는 지연을 허용하기 위한 것이 아니라,  
**지연 여부 자체가 은폐되지 않도록 하기 위한 구조**입니다.

---

## 10. 문서의 범위와 한계

본 백서는 다음을 목적으로 합니다.

- 결과 조작 가능성에 대한 구조 설명
- 공개 검증 방식에 대한 이해 제공
- 누구나 동일한 기준으로 검증 가능하도록 안내

본 문서는 당첨 확률, 성능, 결과를 보장하지 않으며  
투자·예측·보증 문서가 아닙니다.

---

## 11. 마무리

로또아이는 결과를 남기기 전에,  
**결과를 남길 준비 과정부터 외부에 이력으로 남깁니다.**

공개 검증이 특정 서비스의 차별 요소에 머무르지 않고,  
로또 관련 서비스 전반에서  
이용자의 합리적인 선택을 돕는  
신뢰 기준으로 자리 잡기를 기대합니다.
